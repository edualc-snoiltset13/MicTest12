PayPal Login Page Recreation — Plan
====================================

Goal
----
Build a static, educational recreation of the PayPal login page that
looks visually similar to the real product but is clearly labeled as a
demo and never submits credentials anywhere.

Deliverables
------------
1. paypal-login.html  — single-file HTML page (embedded CSS + JS)
2. plan.txt           — this planning document
3. Nav link added to index.html so the demo is reachable

Page Structure
--------------
- Top banner: yellow disclaimer ("UI recreation for educational
  purposes, not affiliated with PayPal, does not submit credentials").
- Centered white card (max-width 460px, 24px radius, soft shadow).
- PayPal-style wordmark rendered as inline SVG text:
    * "Pay" in #003087 (PayPal dark blue)
    * "Pal" in #009cde (PayPal light blue)
- Heading: "Log in to your account".
- Two-step form:
    Step 1 — Email or mobile number (floating label).
    Step 2 — Password field revealed after Next is clicked.
- Primary button: "Next" / "Log In" (#0070ba, hover #003087).
- "Forgot email?" link in PayPal blue.
- Divider with the word "or".
- Secondary outline button: "Create Account".
- Language switcher row: English / Francais / Espanol / Chinese.
- Footer: Contact Us, Privacy, Legal, Policy Updates, Worldwide.

Design Tokens
-------------
Colors
  Primary blue        #0070ba
  Primary hover       #003087
  Light accent blue   #009cde
  Page background     #f5f7fa
  Card background     #ffffff
  Body text           #2c2e2f

Typography
  Stack: "PayPal Sans", "Helvetica Neue", Arial, sans-serif
  Heading 1.25rem / 500
  Body 1rem
  Helper / legal 0.75–0.875rem

Spacing
  Card padding 3rem 3rem 2rem (1.5rem on mobile)
  Input height 56px, button height 48px, button radius 24px

Behavior (client-side only)
---------------------------
- Floating labels animate up on focus or when the field has a value
  (`:not(:placeholder-shown)`).
- Submit handler is preventDefault-only:
    * If on the email step, reveal the password field and rename the
      button "Log In".
    * If on the password step, disable the button and display
      "Demo only — nothing submitted".
- No network calls. No form action. No analytics. No external scripts.

Safety / Non-Phishing Guardrails
--------------------------------
- Persistent yellow banner at the top of the viewport.
- Footer copyright explicitly reads "Recreation. Not affiliated with
  PayPal."
- Form action is omitted; JS never POSTs, fetches, or stores input.
- File lives alongside other demo pages in the repo, not deployed to a
  PayPal-look-alike domain.

Responsive
----------
- Layout is single-column and fluid.
- @media (max-width: 480px): card padding shrinks to 2rem 1.5rem and
  the corner radius drops to 16px.

File Layout
-----------
  /paypal-login.html   new
  /plan.txt            new (this file)
  /index.html          updated to link to the demo page

- Push to origin on the same branch. No PR opened unless requested.
